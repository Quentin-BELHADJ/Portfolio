Tests : Couvrir les chemins critiques pour garantir la fiabilité

Une fois la fonction de connexion développée, j’ai rédigé une série complète de tests unitaires pour valider son comportement dans toutes les situations critiques. L’objectif n’était pas simplement de vérifier qu’elle fonctionne, mais de m’assurer qu’elle réagit correctement, même en cas d’erreurs ou de tentatives inattendues.

Figure 1 : Tests Jest couvrant les cas critiques de la fonction de connexion

La Figure 1 présente cette série de tests automatisés, structurés pour anticiper les comportements à risque et vérifier la robustesse de la fonction. Cette méthodologie m’a été transmise lors des cours de qualité de développement et m’a permis d’appliquer une démarche rigoureuse fondée sur des cas concrets.

Les scénarios testés sont les suivants :

  • Connexion réussie (administrateur et utilisateur)
  • Mot de passe incorrect
  • Nom d’utilisateur invalide
  • Champs requis manquants
  • Erreur interne (ex : base de données indisponible)
  • Tentative d’un administrateur de se connecter via une route utilisateur

Figure 2 : Couverture complète des différents fichiers de l'application

La Figure 2 met en évidence une couverture de test à 100 % sur toutes les métriques clés :

  • Instructions exécutées
  • Branches conditionnelles
  • Fonctions
  • Lignes de code

Tester efficacement, c’est anticiper les failles potentielles avant qu’elles ne provoquent des dysfonctionnements. Cette rigueur fait partie intégrante des bonnes pratiques que j’ai intégrées durant ma formation, en particulier dans le cadre des cours de qualité de développement.

Compétences mobilisées

Cette phase m’a permis de mobiliser plusieurs compétences issues du référentiel du BUT Informatique (page 16) :

  • Réaliser – CE1.04 : concevoir un code fiable, lisible et maintenable
  • Optimiser – CE2.04 : valider les résultats, identifier les erreurs et justifier les choix techniques
  • Optimiser – CE2.05 : contrôler le respect des spécifications via des tests

Cette expérience m’a permis de renforcer mes compétences en qualité de développement et d'appliquer ce que j'ai appris lors de ma formation. Lorsqu’ils sont associés à un workflow GitHub, ces tests automatisés permettent de sécuriser les déploiements en détectant rapidement les erreurs potentielles avant les mises en production.
